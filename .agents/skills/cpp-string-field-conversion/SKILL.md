# C++ String Field Conversion

Use this skill only for converting existing Kotuku object classes from raw `CSTRING`/`STRING` field storage to
`std::string` storage in the class body. Keep the conversion mechanical and local to the target class.

## Reference Pattern

The Config class conversion is the model for this skill:

- TDL field declarations change from `str Field` to `cpp(str) Field`.
- C++ body members change from `STRING`/`CSTRING` pointer-style fields to `std::string`.
- All field array entries change from `FDF_STRING` to `FDF_CPPSTRING` unless there is a good reason for an exception.
- Field getter callbacks use `std::string_view &Value` instead of `CSTRING Value`
- Field setter callbacks use `const std::string_view &Value` instead of `CSTRING *Value`.
- Manual `strclone()` and `FreeResource()` ownership disappears for converted string fields.
- `AC::NewObject` changes to `AC::NewPlacement`, with placement construction of the object body.
- `AC::Free` calls the class destructor after any custom save/release logic has run.

## Workflow

1. Inspect the target class before editing:
   - Read the `.tdl` class definition, the matching `class_*.cpp`, and generated `class_*_def.c` if it is tracked.
   - Search for every use of the target fields, including direct null checks, `FreeResource`, `strclone`, `c_str`,
     `SetField` helpers, and generated public header accessors.
   - Identify non-string manual resources that must remain manually released.

2. Update the TDL source:
   - Convert each relevant field from `str Name` to `cpp(str) Name`.
   - In the class body block, use `std::string Name;` for converted fields.
   - If the converted field is purely virtual and exists only in a hand-written `FieldArray`, there may be no TDL field
     declaration to change; update the field glue and generated public header instead.
   - If the class has inline methods, replace pointer/null assumptions with `empty()`, direct value access, `.assign()`,
     `.c_str()`, and `.size()` as appropriate.

3. Update C++ helper signatures and call sites:
   - Prefer `std::string_view` for internal helper parameters that read string data.
   - Use `std::string_view &Value` for `FDF_CPPSTRING` field getters and `const std::string_view &Value` for setters.
   - For existing getters, return a matching error to the original code if it had an error path for managing "null/unset" behaviour.
   - For setters, assign directly with `Self->Field = Value;` or `Self->Field.assign(Value);`.
   - Replace `if (Self->Field)` and `if ((Value) and (*Value))` with `not Self->Field.empty()` and `not Value.empty()`
     where the old meaning was "has text".

4. Remove converted string ownership code:
   - Delete `FreeResource(Self->Field)` for converted strings.
   - Delete `strclone()` allocation branches for converted strings.
   - Use `.clear()` in reset/clear actions.
   - Keep manual cleanup for raw pointers, handles, buffers, or other resources that are not converted.

5. Fix object lifecycle:
   - Add a placement constructor if it does not already exist:

```cpp
static ERR CLASSNAME_NewPlacement(extClass *Self)
{
   new (Self) extClass;
   return ERR::Okay;
}
```

   - Add `AC::NewPlacement` to the action table.
   - If an existing `AC::NewObject` performs post-construction allocations, subscriptions, or default initialisation that
     requires normal object context, keep it alongside `AC::NewPlacement`; do not move those operations into placement.
   - In the free action, run any class-specific persistence or release logic first, then call `Self->~extClass();`.
   - Do not manually destroy individual `std::string` fields.

6. Update field glue:
   - Change field array entries for converted fields to `FDF_CPPSTRING|...`.
   - Ensure callback prototypes match the new setter/getter signatures.
   - Regenerated public headers should expose `std::string` fields and `std::string_view` custom setters.
   - Generated custom setters for these fields should pass the address of the view to `WriteValue`, matching the
     `FDF_CPPSTRING` type code generated by the IDL tooling.
   - If regenerated public headers revert manual edits, install the affected rebuilt module before rerunning the IDL
     generation step; the generator can read installed class metadata through `origo`.

7. Clean up dependent code:
   - Replace pointer-to-value access caused by converted body members, such as `Field->member` becoming
     `Field.member`, only when that member was intentionally converted to in-place storage.
   - Add `.c_str()` only at API boundaries that still require `CSTRING`.
   - Preserve existing behaviour for empty values versus unset values; do not silently change public semantics.

## Mandatory Checks

Before finishing the conversion, inspect the diff for these issues:

- No converted field is still released with `FreeResource`.
- No converted field is assigned with `strclone`.
- No converted field is tested as a pointer.
- Every `FDF_CPPSTRING` callback signature matches the field array entry.
- The target class has placement construction before any `std::string` member is used.
- Any retained `AC::NewObject` is for non-construction work only; C++ object construction is handled by
  `AC::NewPlacement`.
- The destructor is called exactly once during the class free path.
- Project style rules are followed: use `and`/`or`, use `IS` instead of `==`, avoid `static_cast`, use three-space
  indentation, and keep function arguments in upper camel-case.

## Validation Boundary

Do not run Flute tests, `ctest`, integration tests, or broad behavioural validation at the end of this conversion. The
user manages testing for this task. If repository instructions require a compile check after C++ edits, do only the
minimum affected build needed to satisfy that instruction, and report clearly that test execution was intentionally not
performed.
